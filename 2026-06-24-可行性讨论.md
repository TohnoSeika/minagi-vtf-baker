# VTF 转换工具 — 可行性讨论（2026-06-24）

## 目标

将普通图片转换为求生之路 2（Source 引擎）可用的喷漆 VTF 文件。
完全本地运行，不依赖网络服务。

## 当前流程（来自 Guide.txt）

1. 输入 1024×1024 无透明通道的静态 PNG
2. 打开 https://rafradek.github.io/Mishcatt/
3. 上传图片 → 选择尺寸 1024×1024 → Sampling: Anisotropic → 其他默认
4. 点击 Convert → Save as VTF

## 选定技术方案

**Tauri v2**（前端 + Rust 后端）

- Minagi 已有 Tauri v2 项目经验（参考：`Minagi_File_Classifier_Tauri_Builder_20260616-2.0`）
- 前端用 HTML/CSS/JS（不套框架，和文件分类助手一致）
- 打包体积小，界面精致

## 核心技术栈

| 组件 | 用途 | 说明 |
|------|------|------|
| `image` crate | 图片读取、缩放、通道处理 | Rust 原生，成熟 |
| **VTFLib** | VTF 编码 | C 语言库，需 Rust FFI 绑定 |
| VTFLib 内部 | DXT 压缩（libsquish） | 无需额外处理 |

## 架构

```
前端（Tauri WebView）          Rust 后端（Tauri commands）
  图片预览 ── invoke() ──→    图片读取 + 缩放
  参数面板                    通道处理
  转换进度 ←── event ───      VTFLib FFI → 编码 → 文件输出
```

## 完整处理流程

1. 打开图片（PNG/JPG/BMP 等）
2. 调整尺寸（缩放到目标 VTF 尺寸）
3. 处理透明通道（去透明 / 保留 / 填充背景色）
4. 采样方式（各向异性 / 双线性 / 最近邻等）
5. DXT 压缩（VTFLib 内部调用 libsquish）
6. 输出 .vtf 文件

## VTFLib FFI 绑定

核心工作。VTFLib 的 C API 不大，类似 Minagi 已有项目中的 Win32 FFI 写法：

```rust
extern "C" {
    fn vlCreateVTF(width: u32, height: u32, format: VTFImageFormat) -> usize;
    fn vlSetImageData(handle: usize, mip: u32, data: *const u8, size: usize);
    fn vlSaveVTF(handle: usize, path: *const c_char) -> bool;
    fn vlDestroyVTF(handle: usize);
}
```
